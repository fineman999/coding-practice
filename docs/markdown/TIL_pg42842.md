📝 [PG] 42842 - 카펫 (완전탐색)
1. 문제 핵심 요약

* 조건: 전체 격자(Total) = 갈색(Brown) + 노란색(Yellow)
* 기하학적 특징: 테두리가 갈색이므로, 노란색 가로 = 전체 가로 - 2, 노란색 세로 = 전체 세로 - 2
* 목표: (W - 2) * (H - 2) == Yellow를 만족하는 [W, H] 찾기 (단, W >= H)

2. 언어별 구현 포인트 (TIL 기록용)

* Java:
* OOP 활용: Carpet 객체를 만들어 책임을 분리하고 matches() 메서드로 가독성 확보.
    * 최적화: Math.sqrt()와 final 키워드를 활용한 불변성 강조.
* Python:
* Pytonic: total // h 정수 나눗셈과 total**0.5를 활용한 간결한 표현.
    * 편의성: 별도 클래스 없이도 직관적인 변수 언패킹(w, h = ...)이 강력함.
* Go:
* Idiomatic Go: math 패키지 의존성을 줄이기 위해 h * h <= total 조건 사용.
    * 성능/간결함: 정적 타입의 안정성과 C 스타일의 간결한 루프가 조화로움.

3. 복습하며 깨달은 점

* 공통 최적화: 세로(H)는 최소 3부터 시작하며, 루트(Square Root) 값까지만 확인하면 시간 복잡도를 $O(\sqrt{N})$으로 최적화할 수 있음.
* 수학적 접근: 완전탐색 문제지만 전체 넓이의 약수(Factor) 성질을 이용하면 탐색 범위를 획기적으로 줄일 수 있음.
