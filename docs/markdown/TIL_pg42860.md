## 🧑‍💻 문제 요약

- 이름(name, 최대 20글자)이 모두 'A'로 되어 있고, 각 위치를 원하는 알파벳으로 바꾸고 완성해야 한다.
- 조작 방식:
    - **위/아래(▲/▼):** 현재 칸의 알파벳을 +1/-1, A→B→...→Z→A(순환)
    - **좌/우(◀/▶):** 커서를 한 칸 이동 (끝과 처음으로 순환)

- *각 알파벳과 커서 이동의 총 조작 최소 횟수 구하기.*

---

## 🤔 풀이 아이디어

### 1. 알파벳 변경 비용(▲/▼)

- 각 알파벳마다 `A`에서 해당 글자가 되기 위한 최단거리:
    - 위로: `ord(char) - ord('A')`
    - 아래로: `ord('Z') - ord(char) + 1`
    - **둘 중 더 작은 값 선택**  
      `min(ord(char) - ord('A'), ord('Z') - ord(char) + 1)`

### 2. 커서 이동 최소화(◀/▶)

- 기본: 오른쪽으로만 쭉 가면 `n-1`번 이동
- **A가 연속인 구간을 건너뛰는 경우가 최소를 만들 수 있다**
    - 각 i번째 이후 연속된 A의 끝(next_idx)까지 스킵, 앞으로 되돌아오거나 반대로 점프 등 모든 경우의 최소값 비교
    - 공식(파이썬, 자바/Go 동일):
      ```python
      move_cost = min(
          move_cost,
          2*i + n - next_idx,
          i + 2*(n - next_idx)
      )
      ```

---

## 🚩 코드 예시 (파이썬)

```python
def solution(name: str) -> int:
    n = len(name)
    change_cost = sum(min(ord(c) - ord('A'), ord('Z') - ord(c) + 1) for c in name)
    move_cost = n - 1
    for i in range(n):
        next_idx = i + 1
        while next_idx < n and name[next_idx] == 'A':
            next_idx += 1
        move_cost = min(move_cost, 2*i + n - next_idx, i + 2*(n - next_idx))
    return change_cost + move_cost
```

## 🚩 코드 예시 (Java)

```java
public int solution(String name) {
    int answer = 0;
    int n = name.length();
    // 알파벳 변경 비용
    for (char c : name.toCharArray()) {
        answer += Math.min(c - 'A', 'Z' - c + 1);
    }
    int move = n - 1;
    for (int i = 0; i < n; i++) {
        int next = i + 1;
        while (next < n && name.charAt(next) == 'A') next++;
        move = Math.min(move, 2 * i + n - next);
        move = Math.min(move, i + 2 * (n - next));
    }
    return answer + move;
}
```

## 🧠 교훈 & 리뷰

- 단순히 한 방향으로만 커서를 이동하지 않고,  
  **연속된 'A' 구간을 스킵해서 불필요한 이동을 줄이는 그리디 최적화**가 핵심.
- 아스키 연산, min/max, 인덱스 처리(원형 큐 느낌) 등 기본로직 숙지
- **완전탐색(DFS)로도 가능**하지만 대부분 O(N^2) 이내 구조로 충분
- 실전에서 Java, Python, Go 등 언어마다 문자열/문자 연산 방식 약간씩 다르다는 점을 실감
