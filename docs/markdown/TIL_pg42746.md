## 🐹 1. Go 언어 핵심 지식

### ① `sort.Interface` vs `sort.Slice` 선택 기준
```go
// sort.Interface: 제네릭 구조체에 Len/Less/Swap을 직접 구현
// → 재사용 가능, 타입 안정성, 메모리 효율적
type internalSortBigNumber[T any] struct { ... }
func (s *internalSortBigNumber[T]) Less(i, j int) bool { ... }

// sort.Slice: 클로저로 간결하게 작성
// → 코드 짧음, 일회성 정렬에 적합
sort.Slice(strs, func(i, j int) bool {
    return strs[i]+strs[j] > strs[j]+strs[i]
})
```

| 항목 | `sort.Interface` | `sort.Slice` |
|------|-----------------|--------------|
| 코드 길이 | 길다 | 짧다 |
| 재사용성 | ✅ 높음 | ❌ 낮음 |
| 메모리 | ✅ 적음 (2.2MB) | ❌ 많음 (9.5MB) |
| 속도 | 거의 동일 | 거의 동일 |

### ② `make` + 인덱스 vs `var` + `append`
```go
// ❌ make + append → 길이 2N 버그 (빈 문자열 N개 + 실제 값 N개)
strs := make([]string, len(numbers))
strs = append(strs, "값") // 뒤에 덧붙여짐!

// ✅ make + 인덱스 직접 할당 (재할당 없음, 빠름)
strs := make([]string, len(numbers))
strs[i] = "값"

// ✅ var + append (재할당 있음, 느리지만 안전)
var strs []string
strs = append(strs, "값")
```
- `make([]string, N)`은 이미 길이 N의 빈 슬라이스를 만들므로, `append`를 쓰면 **2N이 됨**!
- `var` + `append`는 내부적으로 용량 부족 시 **2배씩 재할당** → 약 17회 재할당 발생(N=100,000)


### ② `cmp_to_key` 동작 원리
```python
from functools import cmp_to_key

# cmp_to_key는 두 인자 비교함수(cmp)를 단항 key 함수로 변환
# 내부적으로 래퍼 클래스를 만들어 __lt__ 등을 구현
strs.sort(key=cmp_to_key(compare))

# compare(a, b) 반환값의 의미
# -1 (음수): a가 b보다 앞에 와야 함
#  0       : 동등
#  1 (양수): a가 b보다 뒤에 와야 함
```

### ③ `key` 방식 vs `cmp_to_key` 성능 차이
```
key 방식 (x*3):
  Mean ≈ 1.659 ms  ← 빠름
  → 각 원소에 대해 key를 한 번만 계산 (O(N))
  → 이후 비교는 C 레벨에서 수행

cmp_to_key 방식:
  Mean ≈ 8.730 ms  ← 약 5.26배 느림
  → 비교마다 Python 함수 호출 (O(N log N)번)
  → 모든 비교가 Python 레벨에서 수행
```

## ☕ 3. Java 핵심 지식

### ① `Comparator.comparing` 타입 추론 한계
```java
// ❌ .reversed() 체이닝 시 타입 추론 실패
Comparator.comparing(a -> a + a + a + a).reversed()

// ✅ 타입 명시 필요
Comparator.comparing((String a) -> a + a + a + a).reversed()

// ✅ 또는 변수에 먼저 담기
Comparator<String> comp = Comparator.comparing(a -> a + a + a + a);
comp.reversed();
```

### ② `(a, b) -> (b+a).compareTo(a+b)` 가 더 좋은 이유
```java
// ❌ x*4 트릭: 숫자 길이 ≤ 4 제약에서만 안전
Comparator.comparing((String a) -> a + a + a + a).reversed()

// ✅ a+b vs b+a: 길이 무제한, 명확한 의도
Arrays.sort(strs, (a, b) -> (b + a).compareTo(a + b));
```

### ③ `String[]::new` 배열 생성자 참조
```java
// (n) -> new String[n] 과 완전히 동일
```
