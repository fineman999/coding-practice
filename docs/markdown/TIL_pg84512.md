📝 TIL: 알고리즘 문제 풀이 복습 (모음사전)
1. 문제 개요

* 문제: 'A', 'E', 'I', 'O', 'U'만 사용해 만든 5자리 이하 모든 단어 중 특정 단어의 순서 찾기
* 핵심: 사전식 순서(DFS 탐색 순서) 이해 및 효율적인 구현

2. 주요 학습 내용✅ 언어별 구현 연습 (Python, Go, Java)

* Python: global 변수를 활용한 직관적인 재귀 풀이 및 성능 분석.
* Go: 전역 변수 대신 Closure(익명 함수)를 활용해 Thread-safe한 구조 설계.
* Java: 객체지향(OOP) 원칙을 적용하여 데이터(사전)와 로직(탐색기)의 책임 분리.

✅ 효율적인 문자열 처리: String vs StringBuilder

* String (Immutable): 수정할 때마다 힙(Heap) 영역에 새로운 객체를 생성. 재귀/반복문 내 사용 시 메모리 낭비 심함.
* StringBuilder (Mutable): 내부의 가변 크기 배열을 사용해 기존 객체를 수정.
* Backtracking: append() 후 deleteCharAt()을 사용하여 단 하나의 객체로 전체 탐색을 수행하는 최적화 기법 습득.

✅ 알고리즘 복잡도

* 시간 복잡도: $O(5^n)$ ($n=5$일 때 최대 3,905번 연산으로 매우 빠름)
* 공간 복잡도: $O(n)$ (재귀 호출의 깊이가 최대 길이에 비례)

✅ 테스트 및 벤치마크

* Pytest (Python), Testing/Benchmark (Go), JUnit5/JMH (Java)를 활용해 코드의 정확성과 성능을 정량적으로 측정하는 방법 학습.

3. 핵심 인사이트 (OOP & Clean Code)

* SRP(단일 책임 원칙): Solution 클래스에 모든 로직을 넣기보다 Searcher, Dictionary 등으로 역할을 나누면 확장성이 좋아짐.
* 상태 관리: 전역 변수 사용을 지양하고, 메서드 인자나 지역 변수를 활용해 사이드 이펙트 방지.
