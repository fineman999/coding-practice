# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
## 1. 문제 핵심 요약

* 목표: 모든 차량이 최소 한 번은 단속카메라를 만나도록 설치하는 최소 카메라 개수 구하기.
* 핵심 조건: 차량의 진입/진출 지점에 설치되어도 만난 것으로 간주함.

## 2. 알고리즘 설계 (Greedy)
이 문제의 핵심은 "어떤 지점에 설치해야 가장 많은 차를 한 번에 잡을 수 있는가?"입니다.

1. 정렬: 차량이 고속도로를 나가는 지점(Exit)을 기준으로 오름차순 정렬합니다.
* 빨리 나가는 차량부터 순차적으로 처리해야 뒤에 오는 차량들과의 겹치는 구간을 놓치지 않습니다.
2. 카메라 설치:
* 첫 번째 차량이 나가는 지점에 첫 번째 카메라를 설치합니다.
    * 다음 차량의 진입 지점(Entry)이 현재 카메라 위치보다 뒤에 있다면, 기존 카메라로는 이 차를 단속할 수 없습니다.
    * 이때 새 카메라를 해당 차량의 진출 지점(Exit)에 설치하고 개수를 늘립니다.

## 3. 언어별 구현 포인트 (Python, Java, Go)## 🐍 Python

* routes.sort(key=lambda x: x[1])로 간결하게 정렬.
* 슬라이싱이나 for route in routes로 깔끔한 순회 가능.

## ☕ Java

* Arrays.sort(routes, Comparator.comparingInt(a -> a[1]))를 사용.
* Primitive type 처리를 위해 comparingInt를 사용하는 것이 성능상 유리(오토박싱 방지).

## 🐹 Go

* sort.Slice(routes, func(i, j int) bool { return routes[i][1] < routes[j][1] })로 정렬 구현.
* 슬라이스 참조 특성을 이해하고 lastCameraPosition 갱신 시 인덱스 접근 주의.

## 4. 복잡도 분석

* 시간 복잡도: $O(N \log N)$
* $N$개의 차량 경로를 정렬하는 시간이 지배적임. (순회는 $O(N)$)
* 공간 복잡도: $O(N)$
* 정렬 알고리즘에 따른 추가 공간 필요.
