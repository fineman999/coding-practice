## 📝 TIL: 게임 맵 최단거리 (BFS) 복습
## 1. 문제 개요

* 핵심 목표: (0,0)에서 (n-1, m-1)까지 가는 최단 경로 찾기.
* 알고리즘: BFS (Breadth-First Search). 최단 거리를 보장하기 때문.
* 복잡도:
* 시간 복잡도: $O(N \times M)$ (모든 칸을 최대 한 번씩 방문)
    * 공간 복잡도: $O(N \times M)$ (방문 배열 또는 큐 저장 공간)

## 2. 공통 핵심 전략 (Best Practice)

1. 큐(Queue) 활용: 선입선출(FIFO)을 통해 가까운 노드부터 순차적 탐색.
2. 방문 처리 시점 (매우 중요):
* 큐에서 꺼낼 때(pop)가 아니라, 큐에 넣을 때(push/offer) 방문 처리를 해야 중복 방문(Memory Leak/Time Out)을 완벽히 방어할 수 있음.
3. 거리 기록: 별도의 visited 배열 대신 원본 maps 배열에 이전 거리 + 1을 기록하여 공간 절약 및 가시성 확보.
4. 조기 종료: 목적지 (n-1, m-1)에 도달하는 즉시 현재 거리 값을 반환.

## 3. 언어별 구현 노트## 🐍 Python

* collections.deque를 사용하여 $O(1)$의 popleft()를 구현하는 것이 핵심.
* for dx, dy in directions:와 같은 깔끔한 이터레이션 활용.

## ☕ Java

* Record (Java 16+): record Position(int x, int y) {}를 사용하여 불변 객체로 좌표를 관리하면 가독성이 높아짐.
* Queue: ArrayDeque가 LinkedList보다 메모리 및 속도 면에서 효율적임.
* 객체 재사용: 성능이 극도로 중요하다면 int[] 배열이나 x * m + y 인코딩 방식을 고려.

## 🐹 Go

* Queue: 표준 라이브러리에 큐가 없으므로 슬라이스([]T)를 활용.
* 메모리 최적화: queue[1:] 방식은 앞쪽 메모리가 남을 수 있으므로, head 인덱스 변수를 두어 관리하는 방식이 대규모 데이터에서 훨씬 유리함.
* 인덱스 주의: nx < n && ny < m 등 범위 체크 조건문 실수 주의.

## 4. 실수하기 쉬운 포인트 (Checklist)

* 큐에 넣기 전에 방문 처리를 했는가?
* 상하좌우 탐색 시 인덱스 범위(0 <= nx < n)를 정확히 체크했는가?
* 목적지에 도달할 수 없는 경우 -1을 리턴하는 예외 처리를 했는가?
* (Go/Java) 맵의 가로(m)와 세로(n) 길이를 혼동하지 않았는가?
