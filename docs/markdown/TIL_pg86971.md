# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
1. 💡 핵심 알고리즘 및 풀이 전략

* 문제 유형: 완전 탐색 (Brute Force) + 그래프 탐색 (BFS/DFS)
* 핵심 아이디어:
1. 전선(Edge) 리스트를 하나씩 순회하며 해당 전선을 끊었을 때를 가정한다.
    2. 끊어진 한 지점을 시작점으로 하여 연결된 송전탑의 개수를 센다($count_1$).
    3. 반대편 네트워크 크기는 $n - count_1$이 된다.
    4. 두 전력망의 차이 $|count_1 - (n - count_1)|$의 최솟값을 갱신한다.
* 최적화 전략:
* 전선을 실제로 삭제/복구하는 대신, 탐색 함수(bfs)에 ignoreNode 인자를 넘겨 해당 경로를 차단하는 것이 효율적이다.

2. 🚀 언어별 주요 문법 및 학습 포인트🐍 Python (파이토닉한 접근)

* defaultdict(list)를 활용해 인접 리스트를 쉽게 생성.
* 고급 기법: 한 번의 DFS 순회로 서브트리의 크기를 미리 계산하면 $O(N)$으로 풀이가 가능하다.
* abs(n - 2 * subtree_size) 공식을 통해 간결하게 최솟값 산출.

🐹 Go (효율적인 로직 구성)

* 타입 시스템: map[int][]int 혹은 [][]int를 매개변수로 넘길 때 포인터 없이도 참조로 동작함.
* 슬라이스 큐: 별도의 큐 라이브러리 없이 queue = queue[1:] (Pop)와 append를 이용해 가벼운 BFS 구현.
* 정수 절대값: math.Abs는 float64 전용이므로, 정수 연산 시에는 if문으로 직접 처리하는 것이 Go다운 방식.

☕ Java (안전한 자료구조 활용)

* 인접 리스트: ArrayList<Integer>[]보다 List<List<Integer>>가 타입 안전성 면에서 권장됨.
* 비트마스크의 한계: int(32bit)나 long(64bit)은 $N=100$인 경우 범위를 초과하므로 boolean[]이나 BitSet을 사용해야 함.
* BFS 카운트: 시작 노드를 방문 처리할 때 count = 1부터 시작해야 본인 노드가 누락되지 않음.

------------------------------
3. ⚠️ 공통 실수 및 주의사항

    1. BFS 카운팅 누락: 시작 노드를 큐에 넣을 때 count를 1로 초기화했는지 확인.
    2. 범위 초과: 비트마스크 연산 시 데이터 타입의 크기(32/64비트)가 $N$보다 큰지 확인.
    3. 그래프 초기화: 자바나 Go에서 맵/리스트 배열 사용 시 각 요소(내부 리스트)를 반드시 new나 make로 초기화해야 함 (NPE 방지).

------------------------------
📊 성능 비교 요약

* 시간 복잡도: $O(N^2)$ (전선 $N$개 순회 $\times$ 탐색 $N$번)
* 공간 복잡도: $O(N)$ (인접 리스트 및 방문 배열)
