# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
## 1. 문제 개요

* 문제명: [프로그래머스] 정수 삼각형 (Level 3)
* 핵심 개념: 동적 계획법 (Dynamic Programming, DP)
* 목표: 삼각형 꼭대기에서 바닥까지 가는 경로 중 숫자의 합이 최대가 되는 경로 찾기

## 2. 핵심 풀이 전략

* 방식: Bottom-Up DP (바닥에서 위로 올라가는 방식)
* 점점식: triangle[y][x] += max(triangle[y+1][x], triangle[y+1][x+1])
* 최적화: 별도의 DP 테이블을 만들지 않고, 입력받은 triangle 배열을 직접 수정하여 공간 복잡도를 $O(1)$로 최적화 (In-place update)

## 3. 언어별 구현 요약## [Java] - 정석적인 구현

* Math.max()를 사용하여 직관적으로 최댓값을 비교.
* 이중 루프의 인덱스 범위를 length - 2부터 시작하여 불필요한 인덱스 체크(if) 제거.

## [Python] - 간결함과 파이써닉

* range(start, stop, step)를 사용하여 역순 루프를 깔끔하게 처리.
* 인덱스 접근 방식이 가장 효율적이며, 가독성을 위해 zip 활용이 가능하다는 점을 학습.

## [Go] - 실용성과 성능

* math.Max가 float64만 지원하므로, 정수 비교 시 직접 if문을 사용하는 것이 Go다운(Idiomatic) 방식임을 확인.
* 중간 변수(temp) 혹은 인라인 if문을 통해 성능 최적화.

## 4. 복잡도 분석

* 시간 복잡도: $O(N^2)$ (삼각형의 높이를 $H$라 할 때, 전체 원소 수 $N$에 비례)
* 공간 복잡도: $O(1)$ (원본 배열 수정 시 추가 메모리 거의 없음)
