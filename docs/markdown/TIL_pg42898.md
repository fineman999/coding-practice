# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
## 1. 문제 핵심

* 목표: 집(1, 1)에서 학교(m, n)까지 가는 최단 경로의 개수를 구하기.
* 제약: 오른쪽과 아래쪽으로만 이동 가능, 물에 잠긴 지역(웅덩이) 통과 불가.
* 특이사항: 결과값을 1,000,000,007로 나눈 나머지 출력.

## 2. 해결 전략 (DP)

* 점화식: dp[y][x] = dp[y-1][x] + dp[y][x-1]
* 현재 칸까지 오는 방법의 수는 '위쪽에서 오는 경우'와 '왼쪽에서 오는 경우'의 합.
* 예외 처리:
* 웅덩이인 경우 해당 칸의 값을 0으로 처리하여 경로 수 합산에서 제외.
    * 배열 인덱스 밖(경계선) 처리를 위해 n+1, m+1 크기의 배열을 생성하여 구현 편의성 증대.

## 3. 언어별 구현 경험 및 피드백## 🐍 Python

* 특징: 슬라이싱과 인덱싱이 자유롭지만, -1 인덱스가 리스트의 마지막 요소를 가리키는 특성이 있어 y > 0과 같은 경계 검사가 필수적임.
* 복잡도: 시간 $O(NM)$, 공간 $O(NM)$.

## 🐹 Go

* 특징: 2차원 슬라이스 생성 시 make를 중첩 사용하는 구조가 독특함. 정수 연산 시 오버플로우 방지를 위해 합산 과정에서 나머지 연산(%)을 수행.
* 수정: graph[y][x] += ... 대신 = 대입 연산자를 사용하여 코드의 명확성을 높임.

## ☕ Java

* 특징: JUnit 5와 AssertJ를 이용한 유닛 테스트, JMH를 이용한 벤치마크 측정을 통해 코드의 안정성과 성능을 정량적으로 확인.
* 복잡도: $O(NM)$ 공간 복잡도를 사용했으나, 이론적으로 일차원 배열을 이용한 $O(M)$ 최적화가 가능함을 학습.

## 4. 공통 실수 및 주의사항

* 좌표 체계: 문제의 (m, n)은 (가로, 세로)이므로 배열 인덱스 [y][x]로 변환 시 순서가 뒤집히는 점에 주의해야 함.
* 공간 복잡도 오해: $N \times M$ 크기의 2차원 배열을 생성하면 공간 복잡도는 $O(1)$이 아니라 $O(NM)$임.
* 나머지 연산: 최종 결과뿐만 아니라 합산 과정에서 나머지 연산을 해줘야 효율성 테스트와 오버플로우 문제를 방지할 수 있음.
