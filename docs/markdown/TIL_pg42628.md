# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
------------------------------
## 1. 핵심 알고리즘: Lazy Deletion (지연 삭제)

* 개념: 두 개의 힙(Min-Heap, Max-Heap)을 운영하되, 한쪽에서 삭제된 요소를 즉시 반대쪽 힙에서 제거하지 않고 ID(인덱스)를 기록해두었다가, 나중에 힙의 루트(Peek)에 나타날 때 제거하는 방식입니다.
* 장점: 삭제 연산마다 반대편 힙을 뒤질 필요가 없어 모든 연산을 $O(\log N)$에 처리할 수 있습니다.

## 2. 언어별 구현 포인트## 🐍 Python

* heapq 모듈은 최소 힙만 지원하므로, 최대 힙은 음수 부호(-)를 붙여 구현했습니다.
* set을 활용해 삭제된 인덱스를 관리하는 방식이 매우 직관적이고 효율적이었습니다.

## 🐹 Go

* container/heap 인터페이스를 직접 구현(Len, Less, Swap, Push, Pop)해야 하는 점이 특징이었습니다.
* 주의점: queue.Pop()이 아닌 반드시 heap.Pop(queue)를 호출해야 힙 재정렬이 발생한다는 것을 배웠습니다.
* 값 타입(struct)을 사용해 메모리 효율을 챙기면서도 포인터 리시버를 통해 힙 상태를 변경하는 구조를 익혔습니다.

## ☕ Java

* TreeMap을 활용해 힙 두 개를 쓰는 것보다 훨씬 간결하게 구현했습니다.
* 전략 1 (Node 방식): 값과 인덱스를 묶어 모든 입력을 고유 객체로 관리.
* 전략 2 (Frequency 방식): TreeMap<값, 개수>를 사용하여 중복된 숫자의 빈도수를 관리함으로써 메모리를 최적화하는 기법을 확인했습니다.

## 3. 시간/공간 복잡도

* 시간 복잡도: $O(N \log N)$ - 모든 연산(삽입, 최댓값 삭제, 최솟값 삭제)이 로그 시간 내에 수행됩니다.
* 공간 복잡도: $O(N)$ - 입력된 데이터의 개수만큼 데이터를 저장합니다.
