# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
## 1. 문제 개요

* 문제명: N으로 표현 (Programmers Lv.3)
* 핵심 아이디어: 숫자 $N$을 최소 몇 번 사용하여 대상 숫자(number)를 만들 수 있는지 계산
* 제약 사항: $N$ 사용 횟수의 최솟값이 8보다 크면 -1 반환

## 2. 알고리즘 설계: 동적 계획법 (DP)
이 문제는 단순한 탐색이 아니라, "이전 단계의 결과를 조합해 다음 단계의 결과를 도출"하는 DP의 전형적인 구조를 가집니다.

* DP 정의: dp[i] = 숫자 $N$을 i번 사용하여 만들 수 있는 모든 숫자의 집합(Set).
* 점화식:
* $N$을 $i$번 사용하여 만든 숫자는 $j$번 사용한 뭉치와 $i-j$번 사용한 뭉치를 사칙연산한 결과들의 합집합이다.
    * dp[i] = (dp[j] 연산 dp[i-j]) ∪ {NN...N(i번)} (단, $1 \le j < i$)

## 3. 언어별 핵심 구현 포인트## Go

* map[int]bool을 활용하여 집합(Set) 자료구조를 구현.
* 정적 타이핑 언어로서 집합 간 연산 시 데이터 타입 및 연산 범위를 명확히 관리.

## Python

* set 자료구조와 set comprehension을 활용해 코드를 간결하게 작성 가능.
* 나눗셈 연산 시 // (정수 나눗셈)를 사용하여 문제의 "나머지 무시" 조건을 준수하는 것이 중요.

## Java

* List<Set<Integer>> 구조를 사용하여 단계별 결과 저장.
* HashSet을 통해 중복을 제거하고 contains() 메서드로 결과값의 존재 여부를 $O(1)$에 확인.

## 4. 복잡도 분석

* 시간 복잡도: $O(4^K)$ (여기서 $K=8$)
* 매 단계마다 사칙연산(4종류)이 기하급수적으로 늘어나지만, $K$가 8 이하로 작기 때문에 효율적으로 동작함.
* 공간 복잡도: $O(U)$
* $U$는 8단계까지 생성된 중복 없는 결과값들의 총 개수. 모든 연산 결과를 메모리에 저장해야 함.

## 5. 실수하기 쉬운 포인트 (Self-Feedback)

1. 초기 조건: $N$과 number가 같을 경우 $1$을 즉시 반환해야 함.
2. 나눗셈: 나누는 수(y)가 0이 될 수 없음을 반드시 체크해야 하며, 나머지는 버리고 몫만 취함.
3. 조합의 순서: 뺄셈과 나눗셈은 순서에 따라 값이 다르므로 dp[j]와 dp[i-j]의 모든 조합을 고려해야 함.
