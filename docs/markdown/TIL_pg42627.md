## 📝 TIL (Today I Learned)
## 1. 문제 분석: [PG] 42627 - 디스크 컨트롤러

* 핵심: CPU 스케줄링(SJF - Shortest Job First)과 유사한 우선순위 큐 활용 문제.
* 목표: 모든 작업의 (종료 시각 - 요청 시각) 평균값의 정수 부분 구하기.
* 우선순위: 소요 시간(Short) > 요청 시각(Early) > 작업 번호(Small).

## 2. 언어별 구현 포인트 및 비교

| 특징 | Go | Java | Python |
|---|---|---|---|
| Heap 구현 | container/heap 인터페이스 구현 필요 | PriorityQueue 클래스 사용 | heapq 모듈 (함수형) |
| 우선순위 설정 | Less(i, j int) 메서드 정의 | Comparator.comparingInt 체이닝 | 튜플 (a, b, c) 순차 비교 |
| 데이터 구조 | struct | record (Java 14+) | list 또는 tuple |

## 3. 공통 알고리즘 로직 (Best Practice)

1. 사전 정렬: 모든 작업을 요청 시각 기준으로 먼저 정렬 ($O(N \log N)$).
2. 시간 흐름 제어: time 변수를 사용하여 현재 시점까지 들어온 모든 작업을 힙에 Push.
3. 힙 활용: 힙에서 가장 우선순위가 높은 작업을 하나씩 꺼내 실행 후 time 업데이트.
4. 시간 점프: 만약 힙이 비어있다면, 다음 작업의 요청 시각으로 time을 바로 이동 (효율성 핵심).

## 4. 주의해야 할 실수 (Gotchas)

* 원본 인덱스 보존: 입력 배열을 정렬하면 원본 인덱스가 뒤섞이므로, 객체나 튜플에 원본 인덱스 i를 미리 저장해두어야 함.
* 정수 나눗셈: 평균 계산 시 소수점 이하를 버려야 하므로 Python은 //, Go/Java는 정수형 나눗셈을 사용.
* 시간 복잡도: $O(N \log N)$ (정렬 및 힙 연산), 공간 복잡도: $O(N)$ (데이터 저장).

## 5. 오늘의 소감

"같은 로직이라도 언어마다 힙을 다루는 방식(인터페이스 vs 클래스 vs 모듈)이 다르다는 것을 익혔다. 특히 작업 사이의 공백을 처리할 때 time++ 대신 다음 요청 시각으로 점프하는 것이 성능 최적화의 핵심임을 깨달았다."
