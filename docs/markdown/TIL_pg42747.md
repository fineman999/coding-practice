📝 TIL (Today I Learned)

* 핵심: 과학자의 생산성과 영향력을 나타내는 지표인 H-Index 구하기.
* 정의: $n$편의 논문 중, $h$번 이상 인용된 논문이 $h$편 이상인 $h$의 최댓값.

2. 알고리즘 전략

* 정렬(Sort)이 핵심: 인용 횟수를 내림차순으로 정렬하면, 특정 지점까지의 논문 개수($i+1$)가 곧 "해당 인용 횟수 이상인 논문의 수"가 됨.
* 역전 지점 찾기: 논문의 순위(index)가 인용 횟수보다 크거나 같아지는 순간이 H-Index의 경계선임.

3. 언어별 구현 포인트🐍 Python: 파이토닉한 한 줄 코드

* enumerate와 min, max를 조합하여 선언적으로 해결.
* max(map(min, enumerate(citations, 1)), default=0)
* Lesson: map을 활용하면 루프 없이도 컬렉션 전체에 로직을 일괄 적용할 수 있음.

☕ Java: OOP와 함수형의 조화

* int[]는 기본형이라 Collections.reverseOrder()를 바로 쓸 수 없으므로 오름차순 정렬 후 인덱스를 역으로 계산하거나 스트림 사용.
* Lesson: IntStream.range를 활용하면 인덱스 기반의 함수형 프로그래밍이 가능함. 원본 불변성(Immutability) 유지가 중요.

🐹 Go: 고성능과 간결함

* sort.Slice 또는 최신 slices.SortFunc를 활용한 In-place 정렬.
* Lesson: O(n log n)의 정렬 성능이 전체 복잡도를 결정하며, 테이블 기반 테스트(Table-Driven Test)와 벤치마킹(testing.B)으로 성능 검증이 용이함.

4. 복잡도 분석

* 시간 복잡도: $O(N \log N)$ (정렬 알고리즘에 의해 결정)
* 공간 복잡도: $O(1)$ 또는 $O(N)$ (언어별 정렬 방식 및 복사 여부에 따라 다름)

5. 오늘 얻은 Insight

* H-Index는 단순히 숫자를 세는 게 아니라, 그래프 상에서 정사각형($h \times h$)이 인용 곡선 안에 들어가는 최대 크기를 찾는 기하학적 의미가 있음.
* 파이썬의 map(min, ...) 방식은 이 정사각형 모델을 코드로 가장 잘 표현한 예시임.
