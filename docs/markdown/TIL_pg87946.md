# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습
1. 문제 핵심 요약

* 목표: 주어진 피로도 k로 최대한 많은 던전 탐험하기.
* 제약: 각 던전은 [최소 필요 피로도, 소모 피로도]가 있음.
* 특징: 던전 개수가 최대 8개로 매우 적음 → 모든 경우의 수($8!$)를 다 따져도 된다! (완전 탐색)

2. 해결 전략: DFS (재귀)
   가장 깊은 곳까지 가보며 던전 수를 세고, 다시 돌아와서 다른 경로를 찾는 백트래킹 방식을 사용함.

* 방문 체크: 이미 간 던전은 다시 가지 않도록 표시.
* 최댓값 갱신: 매 재귀마다 count의 최댓값을 유지.

3. 기술적 진화 (Python → Java → Go)✅ 단계 1: 일반적인 방문 배열 (boolean[])

* 가장 직관적인 방법.
* 주의점: 재귀 호출이 끝나면 반드시 visited[i] = false로 되돌려줘야 함 (백트래킹).

✅ 단계 2: 비트마스크 (Bitmask) 활용 🚀
던전 방문 상태를 정수 하나(32비트)의 이진수로 표현하는 방식.

* 체크: (visited & (1 << i)) == 0 (i번째 스위치가 꺼져 있는가?)
* 표시: visited | (1 << i) (i번째 스위치를 켠 상태로 다음 재귀 전달)
* 장점:
1. 공간 복잡도: 배열 생성 없이 int 하나면 끝 ($O(1)$).
    2. 안전성: 값 자체가 복사되어 전달되므로, 다녀온 뒤 방문 해제(false)를 수동으로 할 필요가 없음!

4. 언어별 특징 정리

* Python: itertools.permutations를 쓰면 단 몇 줄 만에 모든 경우의 수 계산 가능 (가장 파이토닉).
* Java: Math.max와 재귀 반환값을 이용해 global 변수 없이 깔끔하게 최댓값 산출.
* Go: testing 패키지의 Benchmark를 통해 비트마스크를 썼을 때의 메모리 할당(Alloc) 감소를 눈으로 확인 가능.

5. 시간 복잡도 계산

* 시간: $O(N!)$ — 던전 순서를 정하는 모든 순열.
* 공간: $O(N)$ — 재귀 호출 스택의 깊이.
