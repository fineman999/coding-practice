# 📝 TIL (Today I Learned) - 알고리즘 문제 풀이 복습

## 1. 문제 핵심 요약

* 목표: 장르별 재생 수 합산 → 장르 내 재생 수 내림차순 → 고유 번호 오름차순 기준으로 최대 2곡씩 추출.
* 핵심 도구: 해시 맵(Hash Map)을 이용한 그룹화, 다중 조건 정렬.

## 2. 언어별 핵심 구현 포인트## 🐍 Python

* collections.defaultdict를 활용해 초기값 설정 번거로움 제거.
* 정렬 팁: key=lambda x: (-x[0], x[1])처럼 음수 부호를 활용해 내림차순/오름차순을 한 번에 처리.

## ☕ Java

* getOrDefault()와 computeIfAbsent()로 if-else 분기 없는 깔끔한 맵 핸들링.
* record를 사용해 불필요한 Getter/Setter 없이 가벼운 데이터 객체(Song) 생성.
* 정렬 팁: s1.play == s2.play ? s1.index - s2.index : s2.play - s1.play 삼항 연산자 체이닝.

## 🐹 Go

* map의 초기값(Zero Value)을 활용해 합산 로직 단순화.
* sort.Slice를 사용한 커스텀 정렬 구현.
* 슬라이싱: songs[:limit]와 같은 슬라이싱 문법으로 루프 조건 간소화.

## 3. 공통 인사이트 (Refactoring)

* 데이터 구조화: 장르 합계용 맵과 장르별 상세 곡 리스트용 맵을 분리하는 것이 효율적.
* 복잡도: 모든 언어에서 전체 곡 수 $N$에 대해 $O(N \log N)$의 시간 복잡도를 가짐 (정렬이 지배적).
* 테스트 & 벤치마크: pytest, JMH(Java), testing(Go)을 통해 로직 검증뿐만 아니라 대량 데이터 처리 성능까지 고려하는 습관을 기름.
