## 1. 파이썬/Go/Java 언어별 순열 & 소수 판별

### 순열(모든 조합) 생성 방법
- **파이썬**:
    - `itertools.permutations`를 사용해 중복 없이 모든 순열 생성 (가장 파이토닉, 내부 C구현으로 빠름)
- **Go/Java**:
    - 주로 재귀 함수를 사용해 직접 순열 생성
    - 각 언어의 set/map에 넣어서 중복 제거

### 소수 판정 최적화
- **2부터 $\sqrt{n}$까지만** 약수 검사 → 시간복잡도 개선
- 파이썬은 `range(2, int(n**0.5)+1)`
- Go/Java는 `for i := 2; i <= int(math.Sqrt(float64(n))); ...` 또는 `for(int i=2; i<=Math.sqrt(n); ...)`
    - 파이썬에서는 `range`가 미포함이므로 `+1`,
    - Go/Java에서는 조건식에 ‘이하(<=)’로 끝 수 포함

## 2. 시간복잡도/공간복잡도

- 모든 조합/순열(길이 1~N) 생성: **O(N! × N)**
- 각 숫자에 대해 소수 판정(최대 $\sqrt{M}$): **O(\#조합 × \sqrt{M})**
- 전체 시간복잡도: **O(K × √M)**  
  (K: 모든 조합의 수, M: 후보 최대값)
- 공간복잡도: **O(K)** (set/map에 모든 후보 저장)

## 3. 파이썬/Go/Java set 사용법 요약

- **파이썬**: `set()`, `add`, 타입 지정은 변수 타입 힌트(`set[int]`)로만
- **Go**: `map[int]struct{}` (set 대체), 값 추가/삭제/검사는 map 사용
- **Java**: `Set<Integer> = new HashSet<>()`

- **Go**: `a + b`, 반복 합칠 때는 `strings.Builder` 권장

## 5. Java 스트림(Stream) 오버헤드와 최적화 팁

- Stream/람다(IntStream 등)는 편리하지만 **람다 객체, 중간 연산, 오토박싱/언박싱, GC Pressure 등으로 반복문보다 실행/메모리 비효율**
- 코딩테스트/성능 민감 구간에선 전통적인 for/if 루프가 훨씬 유리
- 성능에 예민할 땐 **기본 타입, 명령형 반복문** 사용

## 7. 기타 실전 팁

- **알고리즘 구현에선 코드 가독성 + 성능 균형**
- 파이썬은 내장, Go/Java는 직접 구현이 더 유리
- 스트림이나 고급 문법은 가독성이 좋지만 반복이 많아지면 루프가 압도적으로 이득

### 예시 코드(파이썬 기준)

```python
from itertools import permutations

def is_prime(n):
    if n < 2: return False
    for i in range(2, int(n**0.5)+1):
        if n % i == 0: return False
    return True

def solution(numbers):
    combs = set(int(''.join(p)) for r in range(1, len(numbers)+1) for p in permutations(numbers, r))
    return sum(is_prime(num) for num in combs)
```

### 예시 코드(Java 기준)

```java
Set<Integer> seen = new HashSet<>();
permutations("", numbers, seen);
int count = 0;
for (int num : seen) if (isPrime(num)) count++;
return count;
```
