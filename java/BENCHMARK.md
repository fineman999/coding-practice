
# 🚀 Java 알고리즘 벤치마크 가이드 (JMH)

자바는 실행 중에 코드를 기계어로 변환하며 최적화하는 **JIT 컴파일러**의 특성 때문에 단순한 `System.currentTimeMillis()`로는 정확한 속도를 측정할 수 없습니다. 
따라서 자바 공식 성능 측정 도구인 **JMH(Java Microbenchmark Harness)**를 사용하여 정밀한 성능을 측정합니다.

---

## 1. 벤치마크 핵심 어노테이션
알고리즘 순수 연산 속도 측정에 최적화된 설정입니다.

* **`@State(Scope.Thread)`**: ⭐️ 각 쓰레드마다 데이터를 독립적으로 가집니다. (배열 정렬 등 데이터가 변경되는 알고리즘에서 오염 방지)
* **`@BenchmarkMode(Mode.AverageTime)`**: 함수 1회 실행에 걸리는 **평균 시간**을 측정합니다.
* **`@OutputTimeUnit(TimeUnit.NANOSECONDS)`**: 결과 시간 단위를 **나노초(ns)** 또는 마이크로초(us)로 지정합니다.
* **`@Warmup(iterations = 2, time = 1)`**: JIT 컴파일러 예열 횟수입니다. (실제 결과에 포함되지 않음)
* **`@Measurement(iterations = 3, time = 1)`**: 예열 후 **본 측정**을 반복할 횟수입니다.
* **`@Fork(1)`**: 이전 테스트의 영향을 받지 않도록 **완전히 새로운 JVM 프로세스**를 띄워 실행합니다.

## 2. 벤치마크 결과표 분석 (How to Read)

테스트가 종료되면 콘솔 마지막에 종합 결과표가 출력됩니다.

### 📊 결과표 예시
```text
Benchmark                                 Mode  Cnt      Score     Error  Units
p.pg42748.SolutionBenchmark.testSolution  avgt    3   5090.375 ± 348.200  ns/op
p.pg86491.SolutionBenchmark.testSolution  avgt    3  17212.153 ± 320.245  ns/op
```

### 🔍 지표 상세 분석
* **`Benchmark`**: 측정된 클래스 및 메서드 이름입니다. 여러 개를 돌리면 한눈에 비교할 수 있습니다.
* **`Mode (avgt)`**: Average Time, 함수 1번 실행에 걸린 **평균 시간**을 뜻합니다.
* **`Cnt (3)`**: 본 측정을 수행한 횟수(Iterations)입니다.
* **`Score` ⭐️ 핵심**: **실제 소요 시간**입니다. 숫자가 작을수록 빠릅니다.
* **`Error`**: 오차 범위(99.9% 신뢰구간)입니다. 이 수치가 `Score` 대비 작을수록 **"속도가 튀지 않고 늘 일정하게 빠르다(안정적이다)"**는 뜻입니다.
* **`Units (ns/op)`**: 시간 단위입니다. (1회 실행당 걸린 나노초)

### 💡 실무적 해석 가이드 (위 예시 기준)
* `pg42748`은 약 5,090ns (5us), `pg86491`은 약 17,212ns (17us)가 소요되었습니다.
* 10,000개의 데이터를 처리하는데 **0.01밀리초**밖에 걸리지 않았으므로, 둘 다 매우 훌륭하게 최적화된 $O(N)$ 알고리즘입니다.
* 에러(Error) 수치 또한 매우 낮아 GC(가비지 컬렉터)나 메모리 병목 없이 아주 안정적으로 동작하고 있습니다.

## 3. 성능 판단 기준 (시간 단위 스케일)

| 단위 | 변환 | 알고리즘 성능 판단 (N=10,000 기준) |
| :--- | :--- | :--- |
| **`ns` (나노초)** | $10^{-9}$초 | $O(1)$ ~ $O(\log N)$ (단순 수학 연산, 매우 빠름) |
| **`us` (마이크로초)** | $10^{-6}$초 (= 1,000 ns) | $O(N)$ (1차원 배열 순회, 1ms 이하로 우수함 🟢) |
| **`ms` (밀리초)** | $10^{-3}$초 (= 1,000 us) | $O(N \log N)$ ~ $O(N^2)$ (정렬 또는 복잡한 로직, 개선 여부 검토 🟡) |
| **`s` (초)** | 1초 (= 1,000 ms) | $O(N^2)$ 이상 (이중 for문 등, 시간 초과 위험 구간 🔴) |
```
