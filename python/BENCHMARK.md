### 📊 pytest-benchmark 결과표 분석 가이드

파이썬 성능 측정 도구인 `pytest-benchmark`가 출력하는 표의 각 항목이 무엇을 의미하는지, 실무 개발자의 관점에서 어떻게 해석해야 하는지 정리한 문서입니다.

#### 1. 단위 (Unit)
* 표 좌측 상단에 표시되는 **`(time in us)`** 또는 **`(time in ns)`**를 가장 먼저 확인해야 합니다.
    * **`ns` (Nanoseconds):** 10억 분의 1초 (매우 빠름, 단순 연산)
    * **`us` (Microseconds):** 100만 분의 1초 (일반적인 알고리즘 성능)
    * **`ms` (Milliseconds):** 1,000분의 1초 (DB 조회, 무거운 연산)

#### 2. 핵심 지표 (Core Metrics)
| 항목 | 설명 | 실무 해석 가이드 |
| :--- | :--- | :--- |
| **Mean** | 실행 1회당 **평균 소요 시간** | 가장 기본이 되는 전체 성능의 평균치입니다. 다른 알고리즘과 대략적인 속도를 비교할 때 사용합니다. |
| **Min** | 측정 중 **가장 빨리** 끝난 시간 | 외부 방해(OS 작업 등)가 전혀 없을 때 낼 수 있는 '코드의 순수 최고 속도'입니다. |
| **Max** | 측정 중 **가장 늦게** 끝난 시간 | 최악의 경우(Worst Case) 또는 일시적인 시스템 멈춤(GC 등)이 발생했을 때의 속도입니다. |

#### 3. 통계적 신뢰도 지표 (Statistical Reliability)
평균(Mean)은 한 번의 비정상적인 지연(Max)에 크게 왜곡될 수 있으므로, 아래 지표들을 함께 봐야 "진짜 성능"을 알 수 있습니다.

| 항목 | 설명 | 실무 해석 가이드 |
| :--- | :--- | :--- |
| **Median** | 전체 기록을 나열했을 때 **정확히 한가운데(50%)** 기록 (중간값) | ⭐️ **가장 신뢰할 수 있는 평소 속도**입니다. 튀는 값에 영향을 받지 않으므로 실질적인 기준점이 됩니다. |
| **StdDev** | 평균으로부터 떨어진 **표준 편차** | 이 값이 작을수록 "실행할 때마다 속도가 일정하다(안정적이다)"는 뜻입니다. |
| **IQR** | 상위 25% ~ 75% 구간의 **속도 차이** | 가장 일관되게 나오는 기록들의 촘촘한 정도를 보여줍니다. 작을수록 좋은 성능입니다. |
| **Outliers** | 통계적으로 **크게 벗어난 기록의 횟수** (`경미함 ; 심각함`) | 백그라운드 프로세스나 쓰레기 수집(GC) 등으로 인해 성능이 튄 횟수입니다. |

#### 4. 처리량 및 측정 환경 (Throughput & Env)
| 항목 | 설명 | 실무 해석 가이드 |
| :--- | :--- | :--- |
| **OPS** | 초당 함수 처리 횟수 (Operations Per Second) | 1초에 이 코드를 몇 번이나 실행할 수 있는지 보여줍니다. 숫자가 클수록 압도적으로 빠른 코드입니다. |
| **Rounds** | 신뢰할 수 있는 통계를 내기 위해 **프레임워크가 함수를 반복 실행한 횟���** | 개발자가 지정하지 않으면, 프레임워크가 함수의 속도를 보고 알아서 적절한 반복 횟수를 결정합니다. |
| **Iterations** | 1 Round 당 코드를 연속으<br/>로 묶어서 실행한 횟수 | 코드가 너무 빨리 끝나서 시간 측정이 어려울 때만 숫자가 올라가며, 보통은 1입니다. |
